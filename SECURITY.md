# 安全策略

## 报告安全问题

请不要在公开 Issue、Pull Request、提交信息、日志或聊天记录中粘贴 API Key、`RULES_ADMIN_TOKEN`、用户敏感文本或可复现攻击载荷。

发现漏洞、凭据泄露、规则绕过、权限绕过或敏感数据暴露时，请私下联系仓库维护者。报告应包含受影响版本、影响说明和最小复现步骤，但必须移除真实凭据与用户数据。

## 凭据处理

- API Key 与 `RULES_ADMIN_TOKEN` 只允许存在于本地 `.env` 或部署平台的 Secret 管理器。
- `.env`、`.env.*` 和运行时数据已经被 Git 忽略；提交前仍须检查 staged files。
- 一旦凭据出现在聊天、Issue、PR、日志或远程提交中，立即撤销并生成新值；不要仅靠删除文本解决。
- `RULES_ADMIN_TOKEN` 仅由规则维护者保存，不应分发给普通协作者。

## 部署提醒

本仓库默认面向本地开发。对外部署前应增加 TLS、反向代理、真实身份认证、最小权限、日志访问控制和速率限制。不要将开发 API 直接暴露到公网。

## 数据处理

`data/logs/` 与 `data/feedback/` 可能包含运行元数据或用户反馈，应按敏感数据管理，不得提交到 GitHub。
